# 实时会议稳定性、分段与业务展示修复设计

日期：2026-07-15

## 1. 目标与范围

本轮修复用户最新反馈的六项实时会议问题：上下文词被识别成正文、多人始终只有一个发言人、空页面工具栏错位、无效状态控件、短停顿导致碎片化，以及 AI 工具泄漏内部 revision/segment id。

实时会议与导入转写继续保持独立记录、独立会话状态和独立 AI 上下文。本轮不修改导入台账职责，不自动删除或改写历史会议原始数据。

## 2. 已确认根因

1. DashScope corpus 收到了“智能转写、声纹注册、强制对齐”等泛功能词，低信息量音频会复述这些提示词；现有过滤只覆盖会议标题。
2. 当前 8100 进程早于源码启动，OpenAPI 没有 `/v1/speakers/embedding`，但健康接口仍只按模型加载成功报告 voiceprint ready，导致所有片段进入同一个降级身份。
3. 编辑器实际有四个纵向区域，CSS 只声明三行网格，空会议时富文本工具栏被 `1fr` 推到中间。
4. “已保存”是静态文案，没有真实保存状态流；空转写时搜索和发言人筛选也没有操作对象。
5. 原生实时流硬编码 `silence_duration_ms=400`，忽略前端会议配置，普通思考停顿会被切成新片段。
6. AI 结果渲染把后端审计字段 `sourceTranscriptRevision/sourceSegmentIds/sourceRanges` 直接显示给业务用户。
7. 前端只用 `realtimeRunning` 防重。WebSocket 处于 CONNECTING 时可再次启动，旧 socket 的 close 回调还会清理新 socket，造成同一会议多个 session 并发落库、重复片段和说话人跟踪重置。

## 3. 方案

### 3.1 上下文词与历史污染

- 实时 corpus 保留人名、机构名和业务专有词，排除泛功能标签。
- 最终 ASR 文本若仅由当前 corpus 词按顿号、分号等拼接而成，则作为 context echo 拒绝落库；正常句子中提到这些词时不做字符串删除。
- 对历史已污染片段采用非破坏性过滤：普通详情展示和 AI 输入跳过明确的纯 context echo，数据库原始记录保留用于审计。

### 3.2 单会议单活会话

- 前端新增 connecting 状态，连接中和运行中都禁用重复启动。
- 每个 socket 捕获自己的 token；只有当前 socket/token 才能更新全局状态或停止麦克风。
- 后端增加 meeting 级进程内 lease。后建立的连接取代旧连接，旧连接收到的 final 不再落库；断开时使用 compare-and-release，不能释放新连接的 lease。
- 进程内 lease 适用于当前单 worker 部署；未来多 worker 部署必须迁移到 Redis 或数据库租约。

### 3.3 会议分段

- `silence_duration_ms` 默认使用 1200ms，并读取前端 realtime config；在供应商合法范围内钳制。
- 最大短语时长继续作为兜底，不因本次修改取消。
- 1200ms 面向会议发言，允许短思考停顿留在同一片段；用户主动暂停或结束仍立即 flush。

### 3.4 说话人与能力健康

- 模型服务健康信息必须显式报告 embedding 能力；业务后端不能再把“模型可加载”等同于“embedding API 可调用”。
- 启动/验收必须同时检查 `/v1/speakers/embedding` 路由并用真实 WAV 做一次 embedding 探针。
- embedding 暂时不可用时保留文本并显示稳定匿名身份，但必须把能力降级暴露为诊断状态，不能伪装多人识别正常。

### 3.5 前端布局与控件

- 编辑器网格固定为：工作台栏、编辑工具栏、转写区、播放器四行。
- 没有转写时隐藏搜索和发言人筛选；只有多于一个发言人时显示筛选。
- 删除没有真实状态来源的“已保存”静态文案。
- 历史纯上下文回声不显示在逐字稿和发言人统计中。

### 3.6 AI 工具

- 摘要、纪要、待办、规整、标记统一禁止展示 revision、segment id、source range 和 `rt-rec-*`。
- 后端继续保留上述字段做可追溯性和陈旧检测。
- 用户界面只在结果已过期时显示中文业务提示；版本选择仅显示模板名和人类可读版本号。

## 4. 验收标准

1. 新实时会议不能生成“智能转写；声纹注册；强制对齐。”片段，历史同类片段不进入 AI。
2. 同一会议重复点击开始最多只有一个可落库 session，旧 socket 关闭不影响新 session。
3. 默认静音端点为 1200ms，约 400ms 的思考停顿不立即新开框。
4. 模型服务缺失 embedding 路由时健康检查明确失败；最新服务启动后真实 embedding 探针成功。
5. 空会议工具栏位于顶部，搜索/筛选不占空态空间，假保存状态消失。
6. 五个 AI 工具均不显示内部 ID 或 revision。
7. 前端静态测试、Python 全量测试、系统 smoke 和浏览器桌面验证全部通过。

## 5. 数据安全

- 不自动删除用户历史会议或片段。
- 不为了修复展示问题篡改原始审计字段。
- 不将声纹 embedding 发送到浏览器或写入会议 JSON。
- 任何临时文件清理只允许使用完整路径逐个删除，禁止递归和批量删除。
